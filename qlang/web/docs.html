Was ist QLANG?

Die Programmiersprache fuer KI-Systeme

QLANG ist eine Graph-basierte Sprache, mit der KI-Systeme miteinander kommunizieren, neuronale Netze bauen, komprimieren und ueberall deployen koennen.

Stell dir vor: Zwei KIs muessen zusammenarbeiten. Normalerweise wuerden sie Text hin- und herschicken (langsam, fehleranfaellig). Mit QLANG senden sie stattdessen Berechnungsgraphen - 47x effizienter, fehlerfrei, und sofort ausfuehrbar.

1 Sprache

KIs senden Graphen statt Text. 3 KB statt 50 KB pro Nachricht. Keine Parse-Fehler moeglich.

2 Kompression

IGQK komprimiert Modelle 16x (Float32 zu ternaer). 918 KB wird zu 59 KB.

3 Compiler

Ein Modell, 9 Ziele: Browser (WASM), GPU, Native, ONNX, Assembly.

4 Agenten

Lokale LLMs (Ollama) arbeiten autonom zusammen ueber das QLANG Protokoll.

Ziel der Sprache

QLANG loest drei Probleme:

Problem 1: KIs koennen nicht effizient kommunizieren

Wenn zwei KI-Systeme zusammenarbeiten wollen, schicken sie sich Text. Das ist wie wenn zwei Ingenieure sich Bauplaene per Telefon beschreiben statt sie zu faxen.

Ohne QLANG:  "Bitte erstelle ein neuronales Netz mit 784 Eingaengen,
              128 versteckten Neuronen, ReLU Aktivierung..."
              = 500 Bytes Text, muss geparst werden, kann Fehler enthalten

Mit QLANG:   graph { input x:f32[784]; node h=matmul(x,W); node a=relu(h); ... }
              = 3 KB binaerer Graph, sofort ausfuehrbar, fehlerfrei

Problem 2: ML-Modelle sind zu gross fuer Edge-Geraete

Ein typisches Modell braucht Python + PyTorch (mehrere GB). QLANG komprimiert und kompiliert es zu einer eigenstaendigen Datei.

PyTorch Modell:     918 KB + Python + PyTorch (2 GB Runtime)
QLANG komprimiert:   59 KB, laeuft standalone, kein Python noetig

Problem 3: Deployment ist komplex

Normalerweise musst du fuer jede Plattform neu exportieren. QLANG kompiliert einmal, laeuft ueberall:

GPUWGSL ShaderParallele Berechnung

Komponenten

C qlang-core

Graph, Tensor, Ops, Typen. Die Datenstrukturen der Sprache.

R qlang-runtime

Executor, Autograd, Training, Ollama-Client, WebSocket-Server.

K qlang-compile

Parser, LLVM JIT, WASM, GPU Shader, CLI, ONNX Export.

A qlang-agent

Agent-Protokoll (QLMS), TCP Server, Graph Emitter, Packages.

IGQK Kompression

Informationsgeometrische Quantenkompression - das mathematische Herzsttueck von QLANG.

Was passiert?

Neuronale Netz-Gewichte (Float32, je 4 Bytes) werden auf ternaere Werte reduziert: {-1, 0, +1} (je 2 Bits).

Original:    [0.342, -0.891, 0.012, 0.756, -0.234]  = 20 Bytes
Komprimiert: [  +1,    -1,     0,    +1,    0   ]     =  2 Bytes (2 Bits pro Wert)
Ratio:       10x Kompression

Mathematische Garantie

Jede Kompression traegt eine formale Beweis-Annotation (@proof theorem_5_2), die eine obere Schranke fuer den Genauigkeitsverlust garantiert:

D ≥ (15n/16) / (2β) · log(1 + β · σ²_min)

Das bedeutet: Der Genauigkeitsverlust ist mathematisch begrenzt, nicht zufaellig.

Pipeline benutzen

So benutzt du das Dashboard (die Hauptseite):

Schritt 1: Aufgabe definieren

Schreibe in das Textfeld was die KI bauen soll. Beispiele:

  • "Baue einen Ziffern-Erkenner" (MNIST Klassifikator)
  • "Erstelle einen Muster-Detektor" (Sensordaten)
  • "Trainiere einen einfachen Klassifikator" (allgemein)

Schritt 2: LLM-Modelle waehlen

Architekt: Das LLM das die Netzwerk-Architektur entwirft (Empfehlung: deepseek-r1:1.5b - schnell).

Evaluator: Das LLM das die Ergebnisse bewertet (Empfehlung: llama3.2 - ausfuehrlich).

Schritt 3: Konfiguration

  • Epochs: Wie oft das Netz die Daten sieht. Mehr = besser, aber laenger. 10 reicht fuer einen Test.
  • Dataset: "Synthetisch" fuer schnelle Tests (5 Sek), "MNIST" fuer echte Handschrift-Erkennung.

Schritt 4: Pipeline starten

Klicke "Pipeline starten". Es passiert automatisch:

  1. Architekt wird gefragt (Ollama LLM entwirft die Architektur)
  2. Build (QLANG baut das neuronale Netz)
  3. Training (Backpropagation mit MLX GPU-Beschleunigung)
  4. IGQK Kompression (16x kleiner)
  5. Evaluator wird gefragt (zweites LLM bewertet das Ergebnis)
Was du im Dashboard siehst: Links das 3D-Netzwerk (leuchtet waehrend Training auf), Mitte den Live-Log jeder Kommunikation, rechts die Metriken (Accuracy, Loss, Kompressionsrate).

KI-Agenten

Ein "Agent" ist ein KI-System das selbststaendig Aufgaben erledigt.

Ohne Agenten

Du tippst:  qlang-cli train-mnist --epochs 20
Du tippst:  qlang-cli ollama generate llama3 "Bewerte das"
Du tippst:  qlang-cli ...
= Du machst alles manuell, Schritt fuer Schritt

Mit Agenten

Du sagst:   "Baue mir einen Spam-Klassifikator"

Agent A (Architekt/qwen3): Entwirft 784 → 128 → 64 → 2
QLANG:                     Trainiert, komprimiert 16x
Agent B (Evaluator/llama3): "96% Accuracy, Kompression OK"

= Alles passiert OHNE dich

Das QLMS Protokoll

Agenten kommunizieren ueber das QLANG Message Stream Protokoll:

  • Binaeres Format (Magic Header: QLMS)
  • 3 KB pro Nachricht statt 50 KB Text
  • Jeder Agent deklariert seine Faehigkeiten (Execute, Compile, Compress, Train, Verify)
  • Nachrichten enthalten komplette Berechnungsgraphen, keine Text-Beschreibungen

QLANG Syntax

Ein QLANG-Programm ist ein Berechnungsgraph:

graph classifier {
  input image: f32[1, 784]        // Eingabe: 28x28 Pixel Bild
  input W1: f32[784, 128]         // Gewichte Layer 1
  input W2: f32[128, 10]          // Gewichte Layer 2

  node hidden = matmul(image, W1) // Matrix-Multiplikation
  node activated = relu(hidden)   // ReLU Aktivierung
  node logits = matmul(activated, W2)
  node predictions = softmax(logits)

  // IGQK Kompression mit Beweis-Annotation
  node compressed = to_ternary(W1) @proof theorem_5_2

  output result = predictions
}

Verfuegbare Operationen

OpBeschreibung
add, mul, subElement-weise Arithmetik
matmulMatrix-Multiplikation
relu, sigmoid, geluAktivierungsfunktionen
softmaxNormalisierung fuer Klassifikation
to_ternaryIGQK Kompression zu {-1, 0, +1}
attentionMulti-Head Attention (Transformer)
layer_normLayer Normalization
conv2d2D Faltung (CNN)
ollama_generateLLM Text-Generierung via Ollama

Haeufige Fragen

Kann QLANG ein LLM wie GPT trainieren?

Nein. LLMs brauchen Tausende GPUs und Monate Training. QLANG trainiert kleine, spezialisierte Modelle (Klassifikatoren, Detektoren) in Sekunden.

Brauche ich Python?

Nein. QLANG ist komplett in Rust geschrieben. Kein Python, kein PyTorch, keine Runtime-Abhaengigkeiten.

Was bringt die Kompression?

Ein 918 KB Modell wird zu 59 KB - klein genug fuer einen Browser, ein Handy oder einen Mikrocontroller. Es laeuft ohne Server, ohne Cloud.

Warum brauche ich Ollama?

Ollama ist optional. QLANG kann auch ohne LLMs Modelle trainieren und komprimieren. Ollama wird nur fuer die "KI-entwirft-KI" Pipeline gebraucht (Architekt + Evaluator).

Was ist der Unterschied zu TensorFlow/PyTorch?

TensorFlow/PyTorch sind Trainings-Frameworks fuer grosse Modelle. QLANG ist eine Sprache + Compiler + Kompressor fuer kleine, deploybare Modelle. Sie ergaenzen sich: Du kannst ein PyTorch-Modell als ONNX exportieren und mit QLANG komprimieren.

Laeuft QLANG auf Windows/Linux?

Ja, aber ohne MLX GPU-Beschleunigung (nur auf Apple Silicon). Die CPU-Version funktioniert ueberall wo Rust kompiliert.
